# About 섹션 카피 리라이트 — 설계 / 크리에이티브 브리프

날짜: 2026-07-01
대상 파일: `components/LenaPortfolio.tsx` (about 섹션 = `#about`, 카피는 `I18N.ko.about` / `I18N.en.about` + `SKILLS`)

## 한 줄 목표

about 섹션의 **문장을 담백하게 다시 쓴다.** 레이아웃은 유지하고, 각 칸이 하는 말과 정보 순서만 살짝 다듬는다. 한국어·영어 둘 다.

## 지금 카피의 문제 (왜 고치나)

현재 제목이 "코드로 학교의 불편을 푸는 학생 개발자" / "A student developer solving school problems with code".
→ **실제로 한 일보다 말이 크다.** 중학교 2학년이 "진짜 문제를 코드로 푼다"고 선언하면 오버스럽고 오히려 힘이 빠진다. 실물은 대단한데(가온케어를 진짜 만들어 씀) 표현이 슬로건 톤이라 신뢰가 깎인다.

## 목소리 (모든 문장의 기준)

1. **실물보다 말이 크지 않게.** "불편을 코드로 푼다" 같은 선언 ❌ → "필요한 걸 직접 만든다" 같은 담백한 사실 ✅
2. **형용사보다 구체.** "더 똑똑한 경험", "끝까지 즐긴다" 같은 뜬 말 ❌ → 실제 한 일(우산 대여를 종이→앱으로)로 증명 ✅
3. **결은 지금 그대로.** 여전히 깔끔한 "학생 개발자" 톤. 갑자기 반말·이모지·귀여운 척 ❌. "지금보다 딱 한 칸 낮춘" 정도.
4. **거짓 없이.** 아래 '사실 원천'에 있는 내용만 쓴다. 없는 경력·수상·수치 지어내기 절대 ❌ (실명·미성년 포트폴리오).

## 구조 (레이아웃 유지, 각 칸 역할만 또렷하게)

레이아웃(제목 → 문단 2개 → 스킬칩 → profile.json 카드)은 그대로 둔다. 각 칸이 담을 말:

| 칸 | 지금 | 바꿀 방향 |
|----|------|-----------|
| tag | `$ cat about.md` | 유지 |
| 제목 | 거창한 선언 | 담백한 한 줄 (오버 뺀 버전) |
| 문단 1 | 소개 + "불편을 풀어낸다" | 지금 나 + 실제 만든 것 하나(가온케어)를 담백하게 |
| 문단 2 | "더 똑똑한 경험 / 끝까지 즐긴다" | 어떻게 일하는지(뭘 써서 어디까지) — 뜬 말 빼고 |
| profile 카드 | status/location/focus (버즈워드) | 값을 솔직한 표현으로 (특히 status·focus) |
| 스킬칩 | 8개 (뒤 3개는 채우기 느낌) | 진짜 핵심만 남길지 최종 선택 때 함께 결정 |

## 채울 카피 슬롯 (한/영 각각)

- `heading` — 담백한 제목
- `p1` — 문단 1
- `p2` — 문단 2
- `stat.statusV` / `stat.locV` / `stat.focusV` — profile 카드 값
- `SKILLS` — 유지 vs 핵심만(예: 뒤 3개 정리) 옵션 제시

## 프로세스 ("UX 라이터 서브에이전트")

1. **UX 라이터 에이전트 여러 명 병렬** — 위 목소리로 슬롯별 카피 후보를 각자 씀 (관점 다양화)
2. **에디터 에이전트** — 후보 중 제일 좋은 문장을 골라 다듬고, 목소리 규칙 위반(오버·형용사·거짓) 걸러냄
3. **최종 후보 제시** — 슬롯별 2~3개 최종안을 인아한테 보여주고 **인아가 고름** (한/영)
4. 고른 카피를 `I18N` + `SKILLS`에 반영

## 하지 않을 것 (범위 밖)

- 다른 섹션(hero·work·contact) 카피
